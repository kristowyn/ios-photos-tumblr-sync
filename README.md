# ios-photos-tumblr-sync

A Python script that syncs a macOS Photos album to Tumblr — one post per photo, backdated to the original EXIF capture date, with AI-detected food tags generated by Claude Vision.

## Features

- Posts photos in chronological order (oldest first)
- Backdates each post to the photo's original capture date/time
- Automatically converts timestamps to UTC using GPS location data
- Generates food tags with Claude Vision (`#food`, `#pizza`, `#sushi`, etc.)
- Includes location and camera model in captions when available
- Resumable — tracks posted photos in a local log file so interrupted runs can continue safely
- Handles HEIC (iPhone) photos natively without extra dependencies

## Prerequisites

- **macOS** (required — uses the macOS Photos library)
- **Python 3.11 or later**
- **Photos app access** — Terminal (or your IDE) must be granted access in System Settings
- **Tumblr API credentials** — consumer key/secret + OAuth token/secret
- **Anthropic API key** — for Claude Vision food detection

## Setup

### 1. Clone the repository

```bash
git clone https://github.com/kristowyn/ios-photos-tumblr-sync.git
cd ios-photos-tumblr-sync
```

### 2. Create a virtual environment

```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure environment variables

Copy the example env file and fill in your credentials:

```bash
cp .env.example .env
```

Then edit `.env` with your values (see [Configuration](#configuration) below).

### 5. Grant Photos access

On first run, macOS will show a permission dialog asking whether Terminal may access your Photos library. Click **Allow**. If you miss this dialog or it doesn't appear, go to:

> **System Settings → Privacy & Security → Photos**

and enable access for Terminal (or your IDE).

## Configuration

| Variable | Description | Where to get it |
|---|---|---|
| `TUMBLR_CONSUMER_KEY` | Tumblr app consumer key | [Tumblr API dashboard](https://www.tumblr.com/oauth/apps) |
| `TUMBLR_CONSUMER_SECRET` | Tumblr app consumer secret | Same as above |
| `TUMBLR_OAUTH_TOKEN` | OAuth access token | Generated during Tumblr OAuth flow |
| `TUMBLR_OAUTH_SECRET` | OAuth access token secret | Generated during Tumblr OAuth flow |
| `TUMBLR_BLOG_NAME` | Your Tumblr blog name | e.g. `myblog` (without `.tumblr.com`) |
| `ANTHROPIC_API_KEY` | Anthropic API key | [Anthropic Console](https://console.anthropic.com/) |
| `PHOTOS_ALBUM_NAME` | Name of the Photos album to sync | Must match exactly (case-sensitive) |

### Getting Tumblr API credentials

1. Go to [https://www.tumblr.com/oauth/apps](https://www.tumblr.com/oauth/apps) and create a new app
2. Note the **Consumer Key** and **Consumer Secret**
3. Use the OAuth flow to generate an **OAuth Token** and **OAuth Token Secret** — the [pytumblr README](https://github.com/tumblr/pytumblr#create-a-client) includes a helper script for this

## Usage

```bash
python main.py
```

The script will:
1. Open your Photos library and find the specified album
2. Sort photos oldest to newest by EXIF date
3. For each photo not yet posted:
   - Export to JPEG (HEIC photos are converted automatically)
   - Send to Claude Vision to identify food items
   - Post to Tumblr with caption and tags, backdated to the capture date
   - Wait 6 minutes before the next post

**Re-running is safe.** Already-posted photos are tracked in `posted_photos.log` and will be skipped automatically.

## Caption format

Each post caption follows this format (fields are omitted if not available):

```
April 12, 2019 | Tokyo, Japan | Shot on Apple iPhone 14 Pro
```

## Tags

Each post is tagged with `#food` plus any food items identified by Claude Vision (e.g. `#ramen`, `#sushi`, `#tacos`). Photos where no food is detected are tagged with `#food` only.

## Notes and limitations

- **Tumblr rate limit**: The Tumblr API allows a maximum of 250 posts per day per blog. The 6-minute delay between posts keeps the script safely under this limit (max 240 posts/day).
- **Timezone handling**: If GPS location data is present, the script looks up the timezone of where the photo was taken and converts the timestamp to UTC for accurate backdating. If no GPS data is available, your Mac's local timezone is used as a fallback.
- **HEIC support**: iPhone photos in HEIC format are exported to JPEG directly by osxphotos without requiring the Photos app to stay open.
- **Non-food photos**: Photos with no visible food are still posted — they just won't receive food-specific tags beyond `#food`.
- **Log file**: `posted_photos.log` stores the internal UUID of each posted photo. Do not delete this file if you want to resume a partial sync.
